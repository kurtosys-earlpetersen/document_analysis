# Asset Management Document Benchmarking Project

**Source:** `Document Benchmarking Project.pdf` (in project root). This file is the canonical context for requirements and methodology.

## 1. Introduction

Structured, repeatable methodology for evaluating investor-facing and marketing documents against:

- Industry best practices  
- Regulatory requirements  
- Accessibility standards  
- Communication effectiveness  

## 2. Industry document landscape

### 2.1 Core investor-facing documents

- Fund factsheets  
- Key Information Documents (KID/KIID)  
- Prospectuses  
- Annual and semi-annual reports  
- Marketing brochures  
- Product guides  
- ESG / sustainability reports  
- Performance reports  
- Portfolio holdings reports  
- Client statements  

### 2.2 Marketing document types

- Product overview documents  
- Campaign materials  
- Pitchbooks  
- Thought leadership papers  
- Investor presentations  

## 3. Taxonomy — core components

| Category   | Components |
|-----------|------------|
| Fund overview | Objective, strategy, benchmark, risk profile |
| Performance   | Historical returns, benchmark comparison, charts |
| Portfolio     | Top holdings, sector allocation, geography |
| Fees          | TER, management fees, entry/exit costs |
| Risk          | Risk scale, volatility, drawdowns |
| ESG           | ESG strategy, ratings, impact metrics |
| Disclosures   | Legal, regulatory, risk warnings |
| Operational   | Fund manager info, launch date, ISIN |

## 4. Peer group (example benchmark set)

BlackRock, Vanguard, Fidelity, J.P. Morgan AM, Goldman Sachs AM, PIMCO, Schroders, UBS AM, Amundi, Capital Group.

## 5. Benchmarking matrix (evaluation dimensions)

| Dimension     | Criteria |
|--------------|----------|
| Clarity      | Plain language, readability |
| Transparency | Completeness of disclosures |
| Design       | Layout, hierarchy, visuals |
| Accessibility | Screen reader support, contrast |
| Compliance   | Regulatory alignment |
| Usability    | Navigation, structure |

**Scoring:** 1 = poor, 3 = adequate, 5 = best-in-class.

## 6. Balanced Scorecard (BSC) mapping

| Perspective        | Document metrics |
|--------------------|------------------|
| Customer           | Clarity, usability, accessibility |
| Internal processes | Production efficiency, consistency |
| Learning & growth  | Innovation in communication, ESG integration |
| Financial          | Cost efficiency, scalability |

**Operational metrics:** production cycles, revision frequency, turnaround times.

## 7. Accessibility & assistive technology

### 7.1 Standards

- WCAG 2.1 / 2.2  
- PDF/UA  
- Section 508 (US)  
- EN 301 549 (EU)  

### 7.2 Core requirements

- Proper heading structure  
- Alt text for images  
- Logical reading order  
- High colour contrast  
- Keyboard navigability  
- Tagged PDFs  

### 7.3 Assistive technology compatibility

Screen readers (JAWS, NVDA, VoiceOver), text-to-speech, magnification.

| UCITS         | EU            | Standardised retail disclosures |
| PRIIPs        | EU            | KID requirements |
| SEC           | US            | Disclosure and reporting |

| FSCA          | South Africa  | Local compliance |
| ASIC          | Australia     | Financial product disclosure |

## 9. Evaluation methodology — key dimensions

- Document taxonomy alignment  
- Investor communication clarity  
- Portfolio transparency  
- Performance disclosure quality  
- Visual design and usability  
- Accessibility compliance  
- Regulatory adherence  

## 10. Example analysis framework

**Strengths:** clear objective, strong visual hierarchy, comprehensive disclosures.  
**Weaknesses:** poor accessibility tagging, overly technical language, inconsistent layout.

## 11. Suggested outputs

Radar charts, scorecards, heatmaps for accessibility gaps.

## 12. Key insights (from project)

- Leaders strong on clarity and consistency  
- Accessibility often a gap  
- ESG integration growing  
- Compliance strong but varies by region  

## 13. Recommendations

**Immediate:** accessibility tagging, simplify language, standardise templates.  
**Strategic:** digital-first design, ESG storytelling, better data visualisation.
